# Flow Runner Qt 真实 Windows 环境验收清单

记录测试日期、Windows 版本、Python 版本、显示器数量、DPI 比例、OCR 引擎路径和测试项目路径。每项填写 `PASS`、`FAIL` 或 `BLOCKED`，并附截图或日志路径。

| 项目 | 操作 | 通过标准 | 结果/证据 |
|---|---|---|---|
| 应用启动 | 在项目虚拟环境运行 `flow-runner` | 主窗口打开，无导入异常 | |
| QSS | 检查三栏、选中和运行状态 | 样式来自 `base.qss`，控件无局部硬编码 | |
| 单显示器截图 | 预览 desktop 条件 | 截图尺寸和坐标正确 | |
| 多显示器截图 | 在副屏框选目标 | 区域和绝对坐标正确 | |
| DPI | 分别使用 100%/125%/150% | 框选与点击坐标一致 | |
| Tesseract | 配置可用程序与语言包 | 文本、矩形、置信度可见 | |
| PaddleOCR-json | 配置客户端/进程适配器 | 中文文本和坐标可见 | |
| 图片匹配 | 使用已知模板 | 命中坐标与阈值正确 | |
| 像素检测 | 选择固定颜色点 | 容差内命中，容差外不命中 | |
| 区域变化 | 切换游戏画面 | 连续帧变化比例正确 | |
| 鼠标 | 点击、移动、滚轮 | 按配置执行且停止可中断 | |
| 键盘 | 单键、组合键、文本 | 按配置执行且停止可中断 | |
| 录制回放 | 保存并回放短脚本 | 事件顺序、速度、最大停顿正确 | |
| 全局热键 | 启动、暂停、停止、录制 | 非焦点窗口下仍触发且无重复绑定 | |
| 跨组路由 | 运行 A1→A2→A3 循环后转 B/C | 路径和计数符合规则 | |
| 调用返回 | 主流程调用子流程 | 子流程完成后回到下一步骤 | |
| 并行监控 | 两个只读监控同一目标 | 共享帧，无重复高频截图 | |
| 资源冲突 | 两流程同时点击同一窗口 | 动作串行，旧坐标执行前重检 | |

| 停止退出 | 轮询或等待中停止并关闭 | 有界时间内结束，无残留监听/进程 | |
| 配置恢复 | 人为损坏主 JSON | 报告明确错误，备份仍可恢复 | |
| 原子保存 | 保存时模拟异常 | 原文件不被半写入文件替换 | |

全部必需项为 `PASS` 后，才能宣告真实环境验收完成。
