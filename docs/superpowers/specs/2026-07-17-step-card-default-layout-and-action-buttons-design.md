# 步骤卡片、默认布局与动作按钮设计

## 目标

完善步骤卡片在多路由和窄步骤区域下的显示，固化当前窗口与三栏尺寸为无保存配置时的默认值，并将动作引导的动作类型下拉列表替换为响应式按钮组。

## 步骤卡片

- 步骤列表不显示横向滚动条。
- 卡片内容宽度跟随步骤列表视口的可用宽度。
- 多条路由继续按“一条路由一个显式行段”显示。
- 单条路由超过可用宽度时在卡片内部自动折行，不截断文字。
- 步骤区域宽度变化后重新计算卡片及列表项高度，保证最下面的策略或路由文字完整可见。
- 保持现有步骤编号、名称、检测、动作、策略和路由摘要内容与选择行为不变。

## 默认窗口与三栏尺寸

- 没有本地窗口尺寸偏好时，主窗口默认尺寸为 `1723 × 1102`。
- 该尺寸仍受当前屏幕可用区域约束，不能让窗口超出可用桌面。
- 项目设置中没有有效 `settings.ui_layout.column_widths` 时，三栏默认比例值为 `249 / 259 / 1152`。
- 已保存的本地窗口尺寸优先于默认窗口尺寸。
- 项目中已保存的有效三栏宽度优先于默认三栏宽度。
- 用户后续调整和保存布局的现有持久化行为不变。
- 不通过本需求修改当前用户维护的 `data/project.json`。

## 动作引导按钮

- 用一组可互斥选择的动作类型按钮替换 `ActionEditor` 顶部的动作类型下拉框。
- 按钮按照注册表提供的动作元数据生成，继续使用现有中文能力名称。
- 按钮按动作引导区域的可用宽度自动换行，不使用横向滚动。
- 当前动作类型使用项目现有主题中的选中状态突出显示。
- 点击按钮立即切换对应配置表单，并沿用当前的待提交状态和变更信号语义。
- 加载已有动作时，自动选中对应动作类型按钮并加载其配置。
- 未知动作能力继续通过现有错误区域报告，不静默改成其他动作类型。
- 动作列表以及添加、更新、删除、复制、上移、下移功能和保存语义保持不变。

## 实现边界

- 优先复用项目现有的响应式换行布局和共享 QSS，不引入独立配色或新的持久化格式。
- 卡片宽度与高度刷新逻辑限定在步骤列表和卡片组件内。
- 默认尺寸集中定义，避免测试和实现散落重复的魔法数字。
- 动作能力选择的业务数据仍使用稳定的 capability 字符串；按钮只改变展示和交互控件。

## 错误处理与兼容性

- 无效或缺失的窗口、列宽保存值回退到新默认值。
- 小于默认尺寸的显示器继续使用现有窗口尺寸裁剪逻辑。
- 步骤列表没有有效视口宽度时不计算负宽度，并在控件完成布局后再次刷新。
- 保持项目 JSON 模式不变，旧项目无需迁移。

## 测试与验收

- UI 测试覆盖窄步骤区域中的多路由换行、无横向滚动条和底部文字完整高度。
- UI 测试覆盖步骤区域改变宽度后卡片高度重新计算。
- 窗口测试覆盖无偏好时的 `1723 × 1102` 默认值、屏幕约束和已有偏好优先级。
- 主窗口测试覆盖无项目列宽时的 `249 / 259 / 1152` 默认值及已有项目列宽优先级。
- 动作编辑器测试覆盖按钮生成、互斥选中、切换表单、加载已有动作及原有增删改排序行为。
- 先运行相关 UI 测试，再运行完整测试集和项目现有静态检查。
- 手工验收重点：多路由步骤卡片在不同中栏宽度下无横向滚动且末行完整；动作按钮自动换行且选中状态明确；首次启动默认布局符合确认尺寸。
