# 原分辨率坐标点选与区域框选设计

## 背景

当前检测区域和模板截图通过固定大小的 `RegionSelectionDialog` 展示。捕获图片会缩放并留边，用户需要拖动后再点击“确定”，不能直接按目标原位置选择。鼠标动作的固定坐标只能手工填写，没有点选入口，也没有独立目标字段；固定坐标始终按绝对屏幕坐标执行，因此窗口移动后无法保持窗口内相同落点。

本设计实现用户已从三列响应式界面任务中拆出的第二任务：原分辨率冻结覆盖层、区域松开即完成、鼠标单击点选、Esc 取消、可选隐藏程序界面，以及窗口相对固定坐标。

## 目标

1. 检测区域和模板截图在捕获目标的原位置、原像素画面上框选。
2. 桌面目标展示完整虚拟桌面，窗口目标展示捕获到的完整窗口边界。
3. 区域拖动松开即成功，鼠标坐标单击即成功，Esc 统一取消。
4. 鼠标动作拥有独立目标字段和“点选坐标”入口。
5. 窗口固定坐标保存为目标相对坐标，执行时按窗口最新原点转换。
6. 动态检测结果继续使用现有绝对屏幕坐标，旧项目固定坐标保持兼容。
7. “框选时隐藏程序界面”作为本机偏好保存并用于全部选择操作。
8. 选择成功、取消或异常后都恢复此前可见的本程序窗口。

## 非目标

- 不改变 `$result...` 动态绑定语法或视觉条件结果的绝对坐标语义。
- 不改变 OCR、图片、像素和区域变化条件的捕获、识别或裁剪模型。
- 不把隐藏界面偏好写入项目 JSON。
- 不保存覆盖层大小、位置或选择历史。
- 不增加全局鼠标钩子、Windows Run 操作或后台常驻选点服务。
- 不重新设计三列布局、步骤卡片或其它属性编辑器。

## 方案比较

### 方案 A：目标原位置冻结覆盖层（采用）

捕获目标后，在对应桌面或窗口边界上显示无边框冻结画面。覆盖层按各显示器 DPI 映射捕获像素，选择坐标直接转换到捕获帧坐标。区域释放或点单击后自动关闭。

该方案最符合用户对“完整桌面/完整窗口”和原分辨率选择的要求，也能明确处理捕获帧原点、负坐标副屏和窗口相对坐标。代价是需要独立的覆盖层控制器、显示器映射和完整的恢复路径。

### 方案 B：固定对话框加滚动条和 100% 缩放

实现较小，但用户看到的不是目标在桌面上的实际位置，多显示器和窗口相对坐标不直观，也不满足完整目标原位置展现。

### 方案 C：全局鼠标钩子直接读取坐标

点选实现简单，但区域框选、冻结反馈、DPI、多屏和钩子释放更复杂，且引入不必要的常驻系统输入监听。

## 总体架构

选择系统分成五个单一职责单元：

- `CapturePreferences`：读取和写入 QSettings 中的隐藏界面布尔值。
- `ApplicationVisibilityGuard`：记录当前可见的本程序顶层窗口，临时隐藏并在退出路径恢复。
- `DisplayMapping`：描述 Qt 逻辑屏幕矩形、捕获帧物理像素矩形和设备像素比例之间的映射。
- `NativeCaptureOverlayController`：创建冻结覆盖层、协调跨屏选择、处理完成与 Esc。
- `RegionCaptureService` / `PointCaptureService`：把覆盖层结果转换成检测区域、模板文件或鼠标坐标。

界面编辑器只调用服务并设置字段；捕获、隐藏、DPI 映射和坐标转换不放入 `ModelForm`。鼠标运行时只负责根据已验证配置解析最终点击位置，不依赖 Qt。

## 原分辨率覆盖层

### 捕获输入

覆盖层接收完整 `CapturedFrame`：

- `image` 是目标冻结画面。
- `origin` 是该画面左上角的绝对屏幕物理坐标。
- 图片尺寸定义捕获帧的局部坐标范围 `(0, 0, width, height)`。

桌面捕获使用完整虚拟桌面图片及其可能为负的原点。窗口捕获使用完整窗口矩形，不裁剪为客户区，也不受条件配置中的现有检测区域限制。

### 显示器与 DPI 映射

控制器为捕获帧和每个可用显示器的交集建立 `DisplayMapping`。每个映射保留：

- Qt 逻辑矩形；
- Windows 物理像素矩形；
- 捕获帧内的物理像素裁剪矩形；
- 设备像素比例。

每个显示器片段使用独立无边框、置顶覆盖窗口显示对应冻结画面裁剪，以保持该屏幕上的一个捕获像素对应一个物理显示像素。控制器共享选择起点和当前全局位置；开始拖动后捕获鼠标，使区域可以跨覆盖片段继续。绘制框在每个片段上显示其交集。

若平台无法提供可靠的显示器物理矩形，控制器只允许在能够确定映射的片段上选择，并以中文上下文报告错误，不使用静默的比例猜测。

### 交互

- 点选模式：左键单击后立即返回捕获帧局部点。
- 区域模式：左键按下开始，移动时绘制边框，松开后立即返回非空捕获帧局部区域。
- Esc：取消并返回 `None`。
- 不显示提示栏、确定按钮或取消按钮。
- 过小、越界或不与捕获帧相交的选择不覆盖原表单值。

## 隐藏程序界面

属性面板常用配置顶部显示“框选时隐藏程序界面”。其值由 `CapturePreferences` 使用稳定的应用组织名和应用名写入 QSettings，下次启动自动恢复，不参与项目脏状态。

选择流程为：

1. 记录当前可见的本程序顶层窗口。
2. 若偏好开启，隐藏这些窗口并处理待执行 Qt 事件。
3. 捕获指定目标。
4. 在保持原程序窗口隐藏的情况下显示冻结覆盖层。
5. 用户完成或取消选择。
6. 在 `finally` 中关闭覆盖层并恢复原先可见的窗口和激活状态。

捕获异常、覆盖层创建异常、Esc 和正常完成共享同一恢复路径。原先就隐藏的窗口不得被错误显示。

## 区域和模板数据流

覆盖层始终返回捕获帧内的局部区域。该数据格式与现有视觉条件配置一致：

- “框选区域”直接写入条件的 `region`。
- “框选并截图”先得到同一局部 `region`，再从原始捕获图片裁剪并保存 PNG。
- 只有选择成功后才创建模板目录和文件。
- 取消或异常不创建模板，不修改 `region` 或 `template_path`。

固定大小缩放对话框、letterbox 映射和“确定”按钮全部退出正常选择流程；仅保留必要的纯映射函数用于新的显示器映射测试时才继续存在。

## 鼠标动作配置和点选

### 配置字段

`MouseActionConfig` 增加：

- `target: str = "desktop"`：独立操作目标。
- 内部坐标空间标记，取值为 `screen` 或 `target`，默认 `screen`。

目标在普通表单中显示为“操作目标”。鼠标 `position` 元组编辑器显示“点选坐标”按钮。内部坐标空间标记属于兼容信息，不要求用户在常用配置中操作。

### 点选规则

- `target=desktop`：覆盖完整虚拟桌面；返回点加上捕获帧原点后写为绝对屏幕坐标，坐标空间为 `screen`。
- `target=window:标题` 或带显式捕获模式的窗口目标：覆盖完整窗口；返回捕获帧局部点，坐标空间为 `target`。
- 目标为空时按 `desktop`。
- 点选成功一次性更新坐标和坐标空间，并触发现有待应用状态。
- Esc、捕获失败或点选失败保留原值。

### 手工编辑与动态绑定

旧项目没有新字段时按 `target=desktop`、坐标空间 `screen` 加载，因此原固定绝对坐标不变。

动态 `$result...` 绑定始终按 `screen` 处理，因为视觉条件已把捕获帧局部结果加上帧原点转为绝对屏幕坐标。切换到动态绑定不得叠加窗口原点。普通界面切换回固定窗口点选时再写入 `target` 坐标空间。

手工输入固定坐标不会在未点选时被猜测改写；高级配置中的坐标空间字段明确决定其含义。

## 运行时坐标转换

鼠标动作资源声明包含 `mouse`；窗口目标还包含规范化后的 `window:标题`，从而继续使用现有交互资源锁和场景代次机制。

执行前按以下规则得到最终绝对坐标：

1. `screen` 坐标直接使用。
2. `target` 坐标要求窗口目标；解析窗口当前完整边界原点，将相对点加到最新原点。
3. 对最终点应用现有 `offset` 和抖动规则。
4. 目标窗口不存在、边界无效或 `target` 坐标配合桌面目标时，动作失败并返回带中文上下文的错误。

窗口可以在配置后移动或缩放；每次动作执行都读取最新窗口原点，因此保持窗口内相同相对落点。动态绑定仍由现有绑定解析和陈旧场景重检流程处理。

## 错误处理

- 捕获、显示器映射、窗口查询和模板保存错误在界面显示中文操作上下文，并保留底层错误详情。
- 所有选择退出路径恢复应用可见性和输入捕获。
- 覆盖层不得吞掉正常应用关闭或留下置顶透明窗口。
- 窗口目标找不到时不回退到桌面，避免误点。
- 区域或点越界时不裁剪成看似成功的其它值。
- 模板保存使用现有目录和时间戳命名，不覆盖已有模板。

## 自动化测试

### 纯逻辑

- Qt 逻辑矩形、物理屏幕矩形和捕获帧局部坐标的双向映射。
- 100%、125%、150% DPI 和负坐标显示器。
- 桌面点加帧原点、窗口点保持局部。
- `screen`、`target`、偏移和窗口最新原点的组合。
- 旧鼠标配置默认值和 JSON 往返。

### Qt 界面

- 点选单击立即完成。
- 区域松开立即完成。
- Esc 返回 `None`。
- 跨覆盖片段拖动共享同一选择。
- 隐藏偏好 QSettings 往返。
- 成功、取消和异常都恢复原先可见窗口。
- 鼠标表单点选更新目标相关坐标空间。
- 检测区域和模板表单保持现有序列化格式。

### 引擎和集成

- 窗口移动后固定相对坐标换算到新绝对位置。
- 动态结果绑定不重复叠加窗口原点。
- 桌面旧绝对坐标行为不变。
- 窗口目标参与资源锁。
- 窗口不存在时动作明确失败且不调用鼠标设备。

## 真实 Windows 验收

1. 在 100%、125%、150% DPI 下分别框选桌面区域和点选坐标，确认冻结图像清晰且落点一致。
2. 在含负坐标副屏的虚拟桌面上验证完整桌面覆盖和跨屏坐标。
3. 对指定窗口确认覆盖完整窗口，区域松开即完成、点单击即完成、Esc 取消。
4. 分别关闭和开启隐藏界面偏好，确认选择期间行为正确，重启后偏好保留。
5. 点选窗口相对坐标后移动窗口，再运行鼠标动作，确认命中窗口内相同位置。
6. 验证动态检测结果点击仍命中绝对结果位置。
7. 框选并截图后检查 PNG 像素尺寸和所见区域完全一致。
